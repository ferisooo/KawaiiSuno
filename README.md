# Suno Kawaii Player 🎀

A cute black / pink / yellow desktop music player built with Electron, for your
Suno AI songs. Heavily animated: living gradient background, floating particles,
cursor trail, click ripples, glowing title, shimmering tabs, and a live audio
visualizer.

## Tabs

- **🎵 Library** — every song you've imported. Paste a Suno link, import from your
  Suno playlists, connect your account, or try the Chrome login.
- **📃 Lists** — your own playlists *inside* the app. Make one, then right-click any
  song → add it. Click a playlist to view/play it.
- **🌟 Explore** — Suno opens **inside the player** (a built-in browser pane). Browse,
  then **right-click a song → Import** (or "⬇ Import all here" for the whole page).
  Import is opt-in now — nothing is grabbed automatically.

Use the **◀/▶** arrow in the title bar to hide the sidebar and let the player fill
the screen.

## Logging into Suno

- **Connect Suno** / **🔑 My songs** loads suno.com's login right inside the app.
  You sign in normally (credentials go straight to Suno, never through this app).
  It's remembered next time — you only log in once.
- **🌐 Use Chrome login** *(experimental)* tries to import your suno.com cookies from
  Chrome so you skip typing. Modern Chrome encrypts cookies in a way that often
  can't be imported (app-bound encryption) — if it can't, it tells you and you just
  log in manually once.

Private songs work once you're logged in. Right-click to import the ones you want.

> Suno changes their site often. If imports come up empty it usually means their
> page/API shifted and this needs a small tweak — paste-link still works regardless.

## Live lyrics

When a Suno track carries lyrics, they show in a panel under the player and
auto-scroll with the song, glowing the current line. (No timestamps from Suno =
smooth progress-based scroll rather than exact karaoke timing.)

### Player controls

Shuffle and Repeat glow brightly when active. **Repeat** cycles three ways:
off → repeat all (loops the list) → repeat one (shows a little **1** badge).
The album art floats with a glowing ring (no spinning).


## How to run

1. Extract this zip anywhere (e.g. your Desktop).
2. Double-click **`start.bat`**.
   - First launch installs what it needs (needs internet once) and self-repairs
     the Electron binary if it didn't download. After that it runs offline.
3. The window opens. Paste a Suno song link and hit **Load**, or hit **Import from
   my Suno playlists** / **Connect Suno** to sign in and bring your songs in. Click
   a track to play.

> The UI itself never needs the internet — React is bundled locally. Only the
> first-time dependency install and loading Suno tracks need a connection.

## Making a real installer (.exe)

Double-click **`build-exe.bat`**. It installs electron-builder on demand and
produces an NSIS installer in the `dist\` folder. (electron-builder is *not*
installed by default because it's large.)

## What each file does

| File | What it's for |
|------|----------------|
| `start.bat` | Self-healing launcher. Fixes broken Electron installs, builds the UI, then runs the app. |
| `build-exe.bat` | Builds a Windows installer with electron-builder (installs it on demand). |
| `package.json` | Project + dependency manifest, build scripts, electron-builder config. |
| `main.js` | Electron main process: window, saved state (songs + playlists), the Suno harvester, right-click import, Chrome cookie import — all the filesystem/network work. |
| `preload.js` | The small, safe bridge that exposes a handful of functions to the UI. Nothing else is reachable. |
| `index.html` | App shell. Loads only local files (plus optional Google Fonts with a system fallback). |
| `build.js` | Compiles `renderer/app.jsx` → `renderer/app.js` using esbuild. Skips gracefully if esbuild isn't present. |
| `renderer/app.jsx` | **Source** of the UI (readable). Edit this. |
| `renderer/suno-preload.js` | Runs inside the embedded Suno page to harvest songs safely (no debugger). |
| `renderer/app.js` | Pre-compiled UI that actually loads. Regenerated by `build.js`. |
| `renderer/styles.css` | All the kawaii styling and animations. |
| `renderer/vendor/react.production.min.js` | React, bundled locally (no CDN). |
| `renderer/vendor/react-dom.production.min.js` | ReactDOM, bundled locally (no CDN). |
| `assets/icon.png` / `assets/icon.ico` | App icon. |
| `UPDATE.md` | Short running changelog, updated every change. |
| `README.md` | This file. |
| `LICENSE` | MIT license. |

## Editing the UI

Change `renderer/app.jsx`, then either run `npm run build` or just relaunch with
`start.bat` (it rebuilds every run). The compiled `app.js` is what the app loads.

## Safety notes

- `contextIsolation: true`, `nodeIntegration: false` — the renderer can't touch
  Node directly.
- All network and filesystem access lives in `main.js` and is called over IPC.
- Web requests send a descriptive User-Agent.

## Suno tips

- Paste a public Suno **song** link (e.g. `https://suno.com/song/<id>`) or the id.
- For private songs and playlists, log in (Connect Suno / Chrome login), open them
  in the Explore pane, and right-click → Import.
